# 🚀 Running Claude Code Securely (Free Setup with Ollama + VM)

> A practical, security-focused approach to running AI agents safely for **offensive security, pentesting, and lab automation**.

---

## 👨‍💻 Context (Cybersecurity Use Case)

As a cybersecurity practitioner, executing commands generated by AI (e.g., recon, scanning, scripting) directly on a host machine introduces **real risk**:

* Accidental destructive commands
* Data exposure
* Uncontrolled script execution

This setup is designed to **contain risk** while still leveraging AI assistance.

---

## 🧠 Core Idea

* Run AI model on **host (Windows)**
* Run Claude Code inside a **Kali Linux VM**
* Execute all commands **inside an isolated lab environment**

---

## 🔐 Security Objectives

This architecture aligns with core security principles:

✔ **Isolation** – VM sandbox prevents host compromise
✔ **Least Privilege** – AI does not interact with host OS
✔ **Controlled Execution** – All commands run in a contained lab
✔ **Safe Experimentation** – Ideal for recon, scanning, and testing tools

---

## 🏗️ Architecture Overview

```
[ Claude Code (Kali VM - Attack Box) ]
                 ↓
[ Ollama Server (Windows Host) ]
                 ↓
[ Local / Cloud Models ]
```

### 🔎 Breakdown

* **Kali VM** → Acts as a controlled offensive security environment
* **Ollama Host** → Provides model inference (no direct execution)
* **Network Bridge** → Enables safe communication between VM and host

---

## ⚙️ Prerequisites

* Windows system (host)
* VMware Workstation / Player
* Kali Linux VM (attacker machine)
* Ollama installed on host
* Networking enabled between host & VM

---

## 🧩 Step 1: Install Ollama on Host

```bash
ollama serve
```

Verify API:

```bash
curl http://localhost:11434/v1/models
```

---

## 🤖 Step 2: Configure Model (Low Resource Optimization)

Pull model:

```bash
ollama pull gemma4:e4b
```

Modify context window:

```bash
echo FROM gemma4:e4b > Modelfile
echo PARAMETER num_ctx 65536 >> Modelfile
ollama create gemma4-fast -f Modelfile
```

> ⚠️ Note: Claude Code performs best with larger context (64K+), but this is tuned for constrained hardware.

---

## 🐧 Step 3: Prepare Kali VM (Execution Environment)

Install Ollama CLI inside Kali:

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

---

## 🌐 Step 4: Connect Kali to Host (Critical Step)

Identify host IP:

```bash
ipconfig
```

Example:

```
10.0.0.75
```

Update environment:

```bash
nano ~/.zshrc
```

```bash
# Ollama server path
export OLLAMA_HOST="10.0.0.75:11434"
```

Apply:

```bash
source ~/.zshrc
```

---

## 🧪 Step 5: Launch Claude Code

```bash
ollama launch claude --model gemma4-fast
```

## ⚠️ Performance Analysis (From Testing)

### Environment

* GPU: 4GB VRAM
* RAM: 32GB

### Test Scenario

* `nmap -sn` (ping scan)

### Results

* ⏱ Execution: ~4–5 minutes
* ❌ Output quality: Inconsistent

### 🔎 Observation

* Local models are **not ideal** for:

  * Real-time recon
  * Complex reasoning tasks
* Bottleneck: **VRAM + context size**

### ✅ Recommendation

* Minimum **8–16GB VRAM** for local usage
* Otherwise → use cloud models

---

## ☁️ Step 6: Using Cloud Models (Best Practice)

Switching to Ollama cloud models provides:

✔ Faster execution
✔ Better command accuracy
✔ No GPU dependency
✔ Suitable for real-world workflows

---

## 🔐 Security Advantage (Why This Matters)

This setup ensures:

✔ AI-generated commands execute only in **Kali VM**
✔ Host system remains untouched
✔ Safe for:

* Network scanning
* Enumeration
* Script testing
* Automation workflows

> Even if a command is unsafe or incorrect, it is **contained within the VM**.
